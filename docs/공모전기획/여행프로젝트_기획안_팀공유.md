## 2. 아이디어 카탈로그 (상세)

### 2-1. 스톤마크 (Stonemark) — ★ 본선 메인 추천

> **"발로 두는 바둑. Travel Korea, Discover Yourself."**
> 한국 전체가 한 판의 바둑판. 손이 아닌 발로 돌을 두고, 모인 돌이 집이 되고, 그 집이 곧 나의 자화상이 된다.

- **한 줄 컨셉** · 관광지 방문 자체를 "돌 놓기 + 영역 점령 + 자기 발견"으로 전환한 자기발견형 글로벌 트래블 RPG.
- **타겟** · ① 관광에 관심 없지만 게임을 좋아하는 일상 게이머 ② "진짜 한국"을 찾는 외국인 인바운드 관광객.
- **핵심 게임 루프 (4동사)**
  - **[간다]** 관광지 도착 → 한국관광공사 위치기반 API로 반경 200m GPS 인증
  - **[돌 놓는다]** 그 자리에 돌 1개 영구 표시 + 지역 대표사진 퍼즐 조각 1개 획득
  - **[엽서]** 친구·랜덤·돌에 두기 — "친구 추가" 부담 없는 비동기 따뜻한 연결(외로움 해소)
  - **[돌아본다]** 매일 저녁 오늘의 리포트 + 별점 → PDF·인스타 카드 1장 자동 생성("매일의 자화상")
- **차별화 메커니즘 (다른 여행앱엔 없는 것)**
  - **점령률(Coverage)** · 홈 최상단 "당신의 한국 3.7%" 한 줄. 누구도 평생 다 못 채우는 무한 동력. 전국→시도→시군구 다층.
  - **지역 균형 자동 보정** · 수도권 가중치 0.5, 비주류 시군구 ×2 → "강남보다 인제가 6배 점수" → **숨은 명소 발굴 = 관광공사 미션 자동 달성**.
  - **기풍(자기 발견)** · 방문 카테고리 분포로 속기파(빠르게 많이)·두터운 실리파(깊게 오래)·세력파(비주류 개척) 자동 산정. 질문 안 함, 발자국이 답.
  - **자동 한옥(집)** · 돌이 모이면 지역 양식 한옥 자동 생성(제주 돌집·강원 너와집·안동 종갓집).
- **데이터 활용 깊이 (공모전 핵심 가점)** · 관광정보 + 빅데이터(방문자수) + 카테고리 코드 + LOD + 다국어 풀세트를 **융합해 새 가치(점령률 가중치·기풍·희귀도)를 자동 산정**. 단순 호출이 아님.
- **기술 스택** · iOS(SwiftUI 17+) · Android(Kotlin/Compose) · Supabase(Postgres+Auth+Storage+Realtime) · 한국관광공사 KorService OpenAPI.
- **구현 상태** · **iOS·Android 약 95% 완성**(픽셀 한반도, 돌 색상, 1.5초 "마법의 순간" 연출, 퀘스트 7단계, 단증, 주간 PvP 대국까지 코드 완료). 남은 건 Supabase v2 SQL 2개 실행 + Android 서명 빌드 + 스토어 제출.
- **로드맵** · MVP(26.9 본선) → V1.0(27 상반기, 안드 동시·영문 풀세트) → V2.0(27 하반기, 지역화폐 환전 베타) → V3.0(28, AR 야경모드·글로벌).
- **BM** · 프리미엄 패스(B2C) · 지자체 협업(B2G) · 지역화폐 환전 가맹점(B2B) · IP 라이선싱 · 익명 데이터 인사이트.
- **공모전 맥락** · **2026 관광데이터 활용 공모전 본선, 시연 2026년 9월.** 제안서 1·2부 PDF 완비.
- **자료 위치** · GitHub `yuneunmi814-cmyk/stonemark-app` (로컬 Documents엔 없음 — 클론 필요)

### 2-2. 쉼표 (Comma / stardusk)

- **한 줄 컨셉** · 검색·계획 없이 지금 내 위치에서 가장 가까운 고요한 자연으로 원터치 안내하고, 그곳에서 올려다본 하늘을 감정으로 기록하는 힐링 여행 앱.
- **타겟** · 핫플 말고 한적한 자연을 즉흥적으로 찾는 20~40대 도시인.
- **핵심 기능** · 원터치 큐레이션(스와이프) · 스카이뷰/일반지도 토글 · 지오펜스 도착감지+국산 지도앱 길안내 위임 · 하늘 영상 촬영 → K-Means로 색·감정 자동 태깅 · 스와이프 행동으로 취향 학습(설문 0).
- **차별점** · "입력 마찰 제로" 철학 + 목적지가 아니라 "그곳의 하늘"을 콘텐츠로. 강원도 가중치 내장(RTO 특별상 노림).
- **기술** · iOS(SwiftUI) · Android(Compose) · FastAPI · Supabase(PostGIS).
- **상태** · **iOS·Android 거의 완성, 즉시 출시 가능 수준.** 백엔드 Phase 1~6 완료. 남은 건 콘텐츠 모더레이션 L1~L5 게이트 연결, 스토어 심사.
- **공모전 맥락** · 2026 관광데이터 공모전 출품(웹·앱 부문), 강원 특화.

> **스톤마크와의 관계** · 둘 다 "위치 기반 + 감성 기록"이라 컨셉이 일부 겹친다. 본선에 둘 다 내긴 어렵고, **스톤마크가 게임성·데이터 융합 깊이에서 우위**. 쉼표는 별도 출시 또는 스톤마크의 "감성 기록" 요소로 흡수 가능.

### 2-3. 트래블팩 (TravelPack)

- **한 줄 컨셉** · 여행 준비를 3시간에서 3분으로 — 검증된 코스 추천 + 여행 중 도착 자동 체크인 가이드 모드.
- **타겟** · 계획 세우기 귀찮아하는 자유여행객, "정해진 코스" 따라가고 싶은 사람.
- **핵심 기능** · 지역/기간/테마별 완성 코스(전국 23개 지역·87개 코스) · 가이드 모드(PostGIS 300m 반경 자동 체크인) · 오디오 가이드 · 사용자 코스 무료 공유 마켓 · 유튜브 쇼츠 큐레이션.
- **차별점** · 한국관광공사 TourAPI 공신력 + 가이드 모드 + 4-eyes 에디터 검수.
- **기술** · React Native(Expo) · 관리자 CMS(React) · Express/Prisma/PostGIS · Redis · AWS S3.
- **상태** · **출시 직전(M4).** 백엔드 테스트 88개 통과, **위치기반서비스사업 신고 수리 완료(2026-06-15)**. 남은 건 클라우드 배포 + EAS 빌드 + 스토어 제출.
- **맥락** · 공모전이 아니라 **실서비스 출시용**. v1 무료, v2부터 제휴 수수료·프리미엄 코스(코드 보존).

> **포지셔닝** · 공모전보다 실제 시장 출시에 가까운 "진짜 제품". 본선작과 별개 트랙으로 굴릴 수 있음.

### 2-4. 동행 (Donghaeng)

- **한 줄 컨셉** · 시니어가 음성 한 마디로 주변 화장실·맛집·포토스팟·길안내·음악을 받는 AI 여행 동반자.
- **타겟** · 60세 이상 시니어(1차) + 그 자녀·보호자(2차).
- **핵심 기능** · 음성 통합질의(의도 분류) · PostGIS 근접검색 · 랜드마크 기반 음성 도보내비 · 포토스팟 촬영팁 · 보호자 사진/위치 공유 · 오프라인 번들.
- **차별점** · 음성 우선·큰 글씨 시니어 UX · 정식 공공데이터만(스크래핑 0) · 좌표 미저장(zero-retention) · "설치는 자녀, 사용은 부모" 구조.
- **기술** · React Native(Expo)+네이버지도 · FastAPI+PostGIS+Redis.
- **상태** · 백엔드 ~75%(근접검색·의도분류·도보내비 완성), 모바일 ~45%(STT/TTS/지도 연동됨, 경로 렌더링 TODO). LLM 분기·경로 실API 미연동, 현재 제천만 샘플.
- **맥락** · 명시적 공모전/사업 문서는 없음. 타겟이 뚜렷해 별도 사업 또는 시니어·복지 테마 공모에 적합.

### 2-5. 릴트립 (ReelTrip)

- **한 줄 컨셉** · 유튜브·인스타·틱톡 링크를 붙여넣으면 영상 속 장소를 인식해 동선이 짜인 여행 코스로 변환.
- **타겟** · "릴스 보다 가고 싶은 곳은 많은데 코스 짜기 귀찮은" 20~30대.
- **핵심 기능** · SNS 링크→코스 자동 생성 · 멀티모달 LLM 장소 추출 + TourAPI/카카오 실존 검증(환각 제거) · 드래그 편집 · "안 가던 곳" 롱테일 추천 · 지도/타임라인 뷰.
- **차별점** · 빈 화면이 아니라 "이미 본 콘텐츠"에서 출발 → 진입 마찰 최소.
- **기술** · Next.js 16 · FastAPI · Ollama(키 0개 데모모드) · Leaflet · dnd-kit.
- **상태** · MVP. 링크→코스 변환 핵심은 동작 검증. DB 마이그레이션·인증·작업큐는 미완.
- **리스크** · **Instagram/TikTok 추출은 ToS 제약**으로 캡션 붙여넣기 폴백만 가능(YouTube만 확실). 이전 평가에서 "모객 검증 선행 → 피벗 권고".

> **포지셔닝** · 아이디어는 날카롭지만 SNS 추출 합법성이 사망 리스크. 본선 메인보다는 **스톤마크의 보조 입력 수단**(릴스 링크로 첫 돌 추천)으로 흡수하는 게 현실적.

### 2-6. 촌캉스 (Choncance)

- **한 줄 컨셉** · 내 주변 농촌체험마을을 지도에서 찾고, AI가 1박2일 촌캉스 코스를 짜주고, 바로 예약처로 연결.
- **타겟** · 농촌 관광에 관심 있는 가족·연인·친구 그룹, 웰촌이 불편한 사용자.
- **핵심 기능** · 지도/리스트 마을 탐색 · AI 코스 추천(자연어 입력) · 마을 상세+예약 딥링크 · 북마크 · 교체형 LLM(OpenAI/Claude/Ollama).
- **차별점 (vs 웰촌)** · 정보포털이 아닌 행동 앱(지도 우선) + AI 코스 자동 구성 + 예약 끊김 없는 딥링크.
- **기술** · Next.js 16 · FastAPI+SQLModel · Leaflet · data.go.kr 공공데이터.
- **상태** · MVP ~65%. 3화면 브라우저 검증 완료. Capacitor 앱화·실데이터 전량 로드·카카오맵 미실시.

## 3. 본선 추진 추천

### 왜 스톤마크인가 (6종 비교 결론)

1. **독창성** · "발로 두는 바둑"은 글로벌 최초 컨셉. 포켓몬GO·Strava식 "수집·정복"을 넘어 **자기 발견**이라는 새 동기를 제시 → 심사위원에게 한 문장으로 꽂힘.
2. **데이터 활용 깊이** · 공모전 이름이 "관광데이터 **활용**"이다. 스톤마크는 관광정보+빅데이터+카테고리+LOD+다국어를 **융합해 새 지표를 자동 산정** → 활용 깊이가 6종 중 최고.
3. **완성도** · 이미 iOS·Android 약 95% 구현. 본선까지 남은 2.5개월을 "새로 만들기"가 아니라 "다듬고 증명하기"에 쓸 수 있음.
4. **공모 정렬** · 숨은 명소 발굴·지역 균형·외국인 만족·재방문 — 관광공사 미션에 메커니즘이 직접 대응.
5. **확장성** · 지역화폐 환전·지자체 협업·IP 라이선싱으로 사업화 스토리까지 완비.

### 백업 / 조합 옵션

- **백업 1순위 = 쉼표** · 만에 하나 스톤마크 방향을 접을 경우, 이미 출시급으로 완성돼 있어 즉시 대체 가능.
- **흡수 전략** · 릴트립(SNS 링크 → 첫 돌 추천), 쉼표(감성 하늘 기록 → 기보 보강)는 **스톤마크의 부가 기능으로 합류**시키면 본선 데모가 더 풍성해짐.
- **별도 트랙** · 트래블팩은 공모전과 무관하게 실서비스 출시를 병행. 동행·촌캉스는 보류(다른 테마 공모전 후보).

### 고민 중이라면 — 무엇을 중시하느냐로 결정

| 이걸 가장 중시한다면 | 선택 |
|---|---|
| 독창성 · 심사 임팩트 · 데이터 활용 깊이 (하이리스크 하이리턴) | **스톤마크** |
| 안정적 완성도 · 검증된 감성 UX (로우리스크) | **쉼표** |
| 공모전과 별개로 실제 시장 출시 · 수익화 | **트래블팩** (병행 트랙) |

> 셋 다 본선에 낼 수 있는 수준. 다만 스톤마크는 **이미 95% 완성**이라 9월까지 "새로 만들" 부담이 가장 적고, 공모전 이름("관광데이터 **활용**")에 가장 정확히 꽂힌다는 점이 결정적 이점이다.
