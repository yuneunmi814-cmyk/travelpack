## 1. 지역축제 데이터 점검 (30선 조사 취합)
- **외국인 집계 불가** · 축제 방문객 수는 일부 나오나 *외국인 비율·수는 데이터가 없음*. 검증 가능한 데이터로만 가야 하므로 **전체 규모(검증된 대형)** 기준으로 솔팅.
- **개최일 불확실** · 지자체 예산이 그때그때 집행돼 **개최 여부·날짜가 한 달 전에도 확정 안 되는** 경우 다수. 2026년 일정 미상이 많음 → 여행객이 미리 계획 불가.
- **대응** · 인력 한계상 **1만 명 이상 대형 축제만** 선별. 데이터는 **공공데이터포털 오픈API(지역축제 정보)** 활용(시군구별, 실시간 아님·일부 누락) + 출처 전부 표기(신뢰도 보강).
- **행동 가설** · 외국인은 "축제 보러 지역 방문"이 아니라 **"지역 간 김에 축제 참여"** → 외국인 약 78% 서울 집중 → **서울 거점 + 근교** 전략이 적합.

## 2. RNR 재편 — 개발 분담 → 사업 조직
> 어제 셋이 같은 자료조사를 중복으로 한 게 비효율 → 파트를 나눠 **각자 담당 + 상호 검수**.

| 파트 | 담당 | 범위 |
|---|---|---|
| **마케팅** | **윤은미** | 소비자 행동·구매여정 조사, 유입 전략, SEO, 웹 MVP 구현(백엔드) — 메타·인스타 광고 경험·경영학과 |
| **영업** | **최용우** | 지자체·식당·거래처 제휴 솔팅, 식당 세션 구현(프론트), 타겟 지역 조사 |
| **비즈니스 구조·시스템(PM)** | **전도준** | 수익모델 설계, 큐레이션/노출 구조, 레퍼런스 기능 수집, 백업·디벨롭 조율 (영업·마케팅은 본인이 약하다며 사양) |

- 개발 역할(백엔드 윤은미 / 프론트 최용우 / 시스템·UX 전도준)은 유지되며, **그 위에 사업 역할을 얹는** 형태.

## 3. 비즈니스 모델 — 무료 우선, 수익구조는 설계만
- **윤은미** · 초기엔 **진입장벽을 낮추는 완전 무료**(광고·유료 후순위). 카카오톡식 — 사람 먼저 모으고 수익은 나중에. 초기 광고는 이탈률↑.
- **전도준** · 무료로 가되 **수익 구조는 처음부터 계획**해야. 플랫폼 과다 시대라 자연유입이 어려워(외국인은 대체 플랫폼이 넘침) 광고비·매출을 예측해 계획을 세워야 함. 일부 유료 서비스 병행 설계.
- **결정** · **초기 무료로 진입장벽↓, 비즈니스 모델은 설계는 하되 적용(광고·유료화)은 후순위.**

## 4. MVP 서비스 범위 — 3축(당일치기 외국인)
- 핵심 = **① 축제 ② 교통 ③ 식당.** **숙소는 제외**(당일치기·접근성 중심, 예약·환불·결제가 1위 페인포인트라 부담).
- **큐레이션 UX** · 축제를 노출하고 교통·식당을 **아이콘 드래그로 사용자가 직접 조립**(여행팩). 내 위치 기반 자동.
- **레퍼런스 = 에어비앤비** · 외국인 친화·이미 학습된 UX를 유사하게 만들어 교육 비용 절감.

## 5. MVP 산출물 — 웹 먼저 배포
- **웹페이지 MVP를 먼저 빠르게 배포** → 방문자수·검색(SEO)로 수요 검증(가설검증). **더미 아닌 실데이터**, 구글 애널리틱스 부착.
- 분담 · **축제 섹션 = 윤은미**, **식당 섹션 = 최용우** → 이후 합치기.
- **전국 데이터로 구현**(지역 한정이 오히려 번거로움), **검증은 2개 지역**으로.

## 6. 결정사항 요약
1. **검증 지역 = 고양시 + 공주시** (2곳).
   - 후보 비교: 고양(호수예술축제·K-pop 콘서트) ○ / 공주(백제문화제 100만급) ○ / 인천 송도(서울과 차별성 없음) ✕ / 대구 치맥(100만이나 접근성↓) △ / 인천 펜타포트 락페(K문화 부합 안 됨) ✕.
   - 기준: **K-pop·한국문화 부합 + 서울 접근성 + 외국인 친화.**
2. **RNR 재편** = 마케팅(윤은미)·영업(최용우)·비즈니스 구조/시스템(전도준).
3. **MVP = 축제+교통+식당 3축의 웹**(숙소 제외), 에어비앤비 레퍼런스, 실데이터.
4. **비즈니스 모델** = 초기 무료, 수익구조 설계만 선행.
5. 공공데이터(관광공사 TourAPI·지역축제 오픈API·카카오맵/네이버 식당)로 수급 — *API 활용이 공모전 취지의 핵심*.
