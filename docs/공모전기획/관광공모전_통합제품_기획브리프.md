# 관광 공모전 — 통합 제품 기획 브리프

> **무엇** · 그동안 만든 여행 제품(트래블팩·스톤마크·STARDUST)을 **하나의 본선작으로 통합**하는 전략 + 공모전 배점 저격 설계.
> **왜 통합인가** · 각각은 "절반"이다. 트래블팩=진통제(오게 만드는 힘)인데 차별성이 약하고, 스톤마크=비타민(독창성)인데 "왜 가지?"가 약하다. **합치면 서로의 빈칸을 채운다.**
> **작성** · 윤은미(PM) · **날짜** · 2026-06-25 · **공유** · 최용우(기획), 조력자

---

## 0. 한 장 요약

- **통합 제품** = **트래블팩(완성형 코스 + 가이드 모드)** 골격 위에 **스톤마크(영토 점령 + 감성 기록)** 를 *백그라운드 보상 레이어*로 얹는다.
- **핵심 한 줄** · "**완성형 코스 따라 걷기만 해도, 한국이 내 영토로 칠해지고 하루가 별로 기록되는** 여행 앱."
- **푸는 문제** · 서울 78% 쏠림 → 지방소멸. (SCQA 별도 문서 참조)
- **타겟** · 1차 효율추구 2030 → 2차 가족 → 3차 외국인 (배점상 이 순서가 정답)
- **공모전 적합** · 완성도·독창성·데이터활용·시장성 **4개 배점을 모두** 정조준. STARDUST식 "심사 포인트 박기" 글쓰기로 마감.

## 1. 통합 제품 정의

### 1-1. 두 제품을 어떻게 합치나

| 레이어 | 출처 | 역할 | 사용자가 하는 일 |
|---|---|---|---|
| **① 완성형 코스 (진통제)** | 트래블팩 | 지역·기간·테마로 검증된 코스 즉시 제공, 3분 준비 | 코스 고르기 |
| **② 가이드 모드 (진통제)** | 트래블팩 | 도착 자동 체크인(PostGIS 반경), 다음 목적지 안내, 오디오 가이드 | 따라 걷기 |
| **③ 영토 점령 (비타민)** | 스톤마크 | 체크인할 때마다 돌이 놓이고 점령률↑, 비주류 지역일수록 보상↑ | *아무것도 안 함(자동)* |
| **④ 감성 기록 (비타민)** | 스톤마크 + STARDUST | 하루 동선이 별자리로, 방문지가 감성 엽서/카드로 자동 생성 | *아무것도 안 함(자동)* |

> **설계 원칙** · 사용자는 ①②만 "한다". ③④는 **백그라운드에서 따라온다.** 노력 대비 최고의 결과물 — 이게 1차 타겟의 핵심 니즈다.

### 1-2. 핵심 사용자 흐름

```
[코스 고르기] → [코스 따라 걷기 + 가이드] → (도착마다 자동 체크인)
                                              │
                                              ▼ 전부 자동
   ┌──────────────────────────────────────────────────┐
   │ • 그 자리에 돌 놓임 → 한국 지도에 내 영토 칠해짐        │
   │ • 비주류 지역이면 점령 점수 ×N (지역 분산 유도)        │
   │ • 하루 동선이 별자리로, 방문지가 감성 카드로 자동 기록   │
   │ • 저녁에 "오늘의 여행 리포트" 1장 자동 생성(SNS 공유)   │
   └──────────────────────────────────────────────────┘
```

### 1-3. 게이미피케이션 — K-로컬 지수 & 레벨링 (Gemini안 보정 반영)

**① K-로컬 지수 (단일 지표)**
- 코스 따라 체크인할 때마다 상승. **뻔한 수도권 핫플보다 지방 숨은명소·전통시장·로컬 소상공인 식당일수록 크게 오름**(스톤마크 가중치 로직).
- ⚠️ **헛지표(vanity metric) 방지** — 단순 체크인 수가 아니라 **체류 시간 · 로컬 소상공인 방문**을 반영해 "진짜 로컬 경험"을 측정. → 생활인구 KPI와 직결.
- > 💡 심사포인트: 방문자수 빅데이터를 가중치로 융합 = *데이터활용 20점* 직접 어필.

**② 레벨 칭호 — 앱(재미)과 제안서(중립)를 분리 운용**

| 지수 구간 | 앱 내 칭호(바이럴용) | 제안서 표현(공공 톤) |
|---|---|---|
| 0~10% | ✈️ 초보 여행자 | 로컬 탐험가 Lv.1 |
| 10~30% | 🍜 K-컬처 마니아 | 로컬 탐험가 Lv.2 |
| 30~60% | 🏠 동네 주민 | 로컬 탐험가 Lv.3 |
| 60~90% | 🇰🇷 명예 한국인 | 로컬 탐험가 Lv.4 |
| 90%+ | 👑 찐 로컬 | 로컬 마스터 Lv.5 |

- ⚠️ "명예 한국인 / 한국인 패치"는 공공기관 심사에서 **문화 전유·국뽕 리스크 + 번역 불가** → **제안서엔 중립 표현, 앱 안에서만 밈 칭호**.

**③ 랭킹 — "커버리지 기준"으로 (경쟁이 분산을 돕게)**
- ❌ **총점 랭킹** = "고득점 지역만 효율 파밍" → 우리 핵심(분산)을 역행.
- ✅ **"얼마나 다양한 지역을 갔나(커버리지)" + "이번 주 추천 미점령 지역 달성"** 기준 → 경쟁할수록 안 가본 곳으로 흩어진다.
- **내국인 "전국 점령 랭킹"(토박이 자존심 쟁탈전) = 본선 메인.**
- **외국인 "글로벌 K-로컬 랭킹"(국가대항전) = V2 확장 비전** — 본선엔 외국인 실데이터 실증 불가 + 인바운드 가점 0이라 메인 아님. 단 바이럴 잠재력이 커 로드맵에 명시.
- > 💡 심사포인트: 투트랙 랭킹·칭호 = *독창성 30점* / SNS 공유 바이럴 = *시장성·확장성*.

## 2. 푸는 문제 (SCQA 요약 — 상세는 별도 문서)

- **S** · K-관광 사상 최대(2025년 1,873만, 2030년 3천만 목표).
- **C** · 그러나 **서울 78% 1극 집중** + 관광수지 적자 14.5조 + 지방소멸 89개 지역. 기존 앱은 "인기순"이라 쏠림을 *강화*.
- **Q** · 누구를, 뭘로 움직여야 안 가본 지역에 가고·머물게 할까?
- **A** · **"오게 만드는 진통제(완성형 코스) + 안 가본 지역일수록 보상되는 게임(점령률 가중치)"** = 통합 제품.

## 3. 타겟 (3단계, 사용자 정의 반영)

### 1차 타겟 — "계획은 귀찮지만, 남는 건 있었으면 하는" 2030 효율 추구형
- **Pain** · 릴스·블로그로 가고 싶은 곳은 많은데, 동선 짜고 예약하려면 피로.
- **해결** · 완성형 코스로 준비 3분. 코스대로 걷기만 해도 백그라운드로 영토·감성 기록이 남아 **노력 대비 최고의 여행 결과물**.

### 2차 타겟 — 주말 나들이 책임지는 부모·가족
- **Pain** · 매주 "이번엔 또 어디?" 동선 고민 스트레스.
- **해결** · '아이와 가기 좋은' 테마 코스 원터치 + 방문지마다 돌이 찍히고 점령률이 올라 **아이에게도 동기부여·재미(게이미피케이션)**.

### 3차 타겟 — "진짜 로컬 한국"을 원하는 인바운드 외국인
- **Pain** · 서울·제주 너머 지역을 가고 싶지만 정보·언어 장벽.
- **해결** · 다국어 코스 + 한국을 바둑판 삼아 영토를 넓히는 **글로벌 수집형 RPG** 감성.

> **배점 근거** · 공모전은 인바운드에 가점 0, **지역특화(서울 제외)에 가점 +2 & RTO 특별상**. 따라서 1·2차(내국인·지역분산)가 메인, 외국인은 확장. 사용자님 직관이 배점과 일치.

## 4. 왜 "통합"이 결정적인가 — 진통제 + 비타민

| | 단독일 때 약점 | 통합으로 해소 |
|---|---|---|
| **트래블팩** | "그냥 코스 추천 앱" — 독창성·리텐션 약함 | 게임·기록이 독창성(30점)과 재방문을 만듦 |
| **스톤마크** | "점수 때문에 안 가본 시골을 갈까?"(행동변화 가정 불확실) | 완성형 코스가 *오게 만드는 진짜 이유*, 게임은 따라오는 보상 |
| **STARDUST** | 감성은 좋지만 "그래서 뭘 하지?"가 약함 | 감성 기록을 통합 제품의 *리포트/엽서 레이어*로 흡수 |

> **핵심** · 스톤마크의 가장 큰 리스크(*"게임 점수가 실제 이동을 만드는가?"*)를 트래블팩이 **원천 제거**한다. 사람들은 점수가 아니라 **완성형 코스** 때문에 오고, 게임은 그 위에 얹히는 보상이라 검증 부담이 사라진다.

## 5. 공모전 배점 저격 매핑

### 심사 기준 (2024 = 2026 동일 확정)
- **1차(100)** · 서비스 완성도 **30** / 기획력·독창성 **30** / 공공데이터(TourAPI 필수) 활용 **20** / 시장성 **20**
- **2차 PT(100)** · 사업 확장성·시장성 **30** / 완성도 **25** / 독창성·차별성 **25** / 1차 점수 **20**
- **가점(+2씩)** · ①신보 Start-up NEST ②타 기관 오픈API ③**지역특화(서울 제외)** ④교육 10h+
- **특별상** · 우수 지역특화 서비스 → RTO 기관장상(부산·광주·울산·경기·강원·경남·제주)

### 통합 제품 → 배점 매핑

| 배점 항목 | 통합 제품의 무기 | 강도 |
|---|---|---|
| **완성도 30** | 트래블팩 출시직전(테스트 88개·위치사업 신고완료) + 스톤마크 95% + STARDUST 좌표검역·5겹 모더레이션 | ★★★ |
| **독창성 30** | "코스 따라가면 한국이 내 바둑판이 된다" — 글로벌 최초 결합 | ★★★ |
| **데이터활용 20** | TourAPI(코스·관광지) + 빅데이터(방문자수→**점령 가중치**) + 다국어 — 단순호출 아닌 *융합* | ★★★ |
| **시장성 20** | 트래블팩 BM(제휴 수수료·프리미엄 코스) + 게임 리텐션 | ★★ |
| **가점 +2~** | **특정 지역(예: 강원) 특화 버전** → 지역특화 가점 + RTO 특별상 | ★★ |

> **전략 결정 포인트** · "전국 서비스"는 지역특화 가점을 못 받는다. STARDUST가 **강원(area_code=32) 가중치**로 이미 노린 것처럼, 통합 제품도 **본선 데모를 특정 지역(강원 등)으로 좁히면 +2점 & 특별상**을 동시에 노릴 수 있다. ↔ 전국 확장성과 트레이드오프 → 팀 논의 필요.

### "심사 포인트 박기" 글쓰기 (STARDUST 차용)
제안서 각 기능 설명 끝에 STARDUST처럼 `> 💡 심사 포인트:` 박스를 달아 **이 기능이 어느 배점을 먹는지** 명시한다. (예: "도착 자동 체크인 → PostGIS 실연산 → *완성도 30점 증명*")

## 6. 기존 자산 활용 맵 (새로 만들 것 최소화)

| 통합 제품 기능 | 가져올 자산 | 상태 |
|---|---|---|
| 완성형 코스 추천 + 가이드 모드 | **트래블팩** 전체 | 출시 직전 ✅ |
| TourAPI 동기화(관광지·코스) | 트래블팩 `sync-tourapi.ts` | 라이브 ✅ |

| 비주류 지역 가중치(분산 유도) | 스톤마크 점령 가중치 + STARDUST `_GANGWON_BOOST` 패턴 | ✅ |
| 감성 기록(별자리 동선·엽서·리포트) | **STARDUST** stars/trips + K-Means 감정 | ✅ |
| 좌표 검역·5겹 모더레이션·EXIF | STARDUST `validation.py` | ✅ |
| 다국어(외국인 3차) | 트래블팩 i18n + KTO 영문 API | ◐ |

> 거의 다 이미 있다. **통합 = 새 개발이 아니라 "조립 + 한 껍데기로 묶기"** 가 본질. 본선 10월까지 충분.
