## A. 비판적 검증 요약 (먼저 읽기)

| # | 제안 내용 | 검증 | 반영 |
|---|---|---|---|
| 1 | 하이브리드 추적 = 상시 GPS 대신 **Geofence + Activity Recognition(모션)** | ✅ **정확.** TMap·구글맵의 실제 저전력 원리 그대로 | 채택 |
| 2 | 정지 시 GPS Off, 100m Geofence 진입 시 **저전력 타이머(10분)** 후 1회 핑 | ✅ 합리적. Android는 Geofence **DWELL**(loiteringDelay)로 네이티브 지원, iOS는 앱 타이머로 보완 | 채택 + iOS 보완 명시 |
| 3 | "Fused Location으로 **Doze mode를 회피**" | ⚠️ **부정확.** Doze는 '회피' 대상이 아니라 '호환 설계' 대상. 상시 추적은 못 우회함 | **2모드 아키텍처로 정정**(아래 ④) |
| 4 | (모드 구분 없음) | ➕ **누락.** 내비가 끊김 없는 건 *활성 주행 = 포그라운드 서비스* 덕분 | **가이드 활성=포그라운드 서비스 / 평상시=지오펜스** 분리 |
| 5 | (지오펜스 개수 가정 없음) | ➕ **누락.** OS 한도 = **iOS 20개 / Android 100개**. 전국 관광지 동시 감시 불가 | **현재 위치 기준 가까운 N개 동적 등록(슬라이딩 윈도우)** |
| 6 | (스토어 심사 언급 없음) | ➕ **누락·일정 리스크.** 구글플레이 백그라운드 위치 = **별도 선언+영상 심사**, iOS Always = 사유 심사 | 리스크로 §12에 명시(위치기반서비스 신고 제1175호와 별개) |
| 7 | (권한 거부 대응 없음) | ➕ **누락.** 사용자가 'Always'를 거부하면 기능이 죽으면 안 됨 | **거부 시 폴백**(포그라운드 추적 + 수동 "도착했어요") → 핵심 KPI는 백그라운드 없이도 달성 |
| 8 | (프라이버시 설계 없음) | ➕ **보강 = 가점 기회.** 민감 권한일수록 데이터 최소수집이 무기 | **연속경로 미저장·체크인 이벤트만·익명 집계·6개월 파기**(retention 배치 이미 구현) |
| 9 | 별자리 → **바둑(기풍·기보)** 전면 교체 | ✅ 채택 + **오히려 정합성↑.** 'Stonemark(돌·점령)'·'점령률'·기록(리포트)과 바둑(집/영토·기보·기풍)이 의미로 연결됨(별자리는 점령 게임과 은유 연결이 약했음) | 채택 + 기풍 산정 로직·문화접근성 보강 |

> 💡 **핵심:** 제안의 큰 방향(하이브리드 저전력 추적)은 옳다. 다만 **"Doze 회피"는 과장**이므로 *2모드(활성=포그라운드 서비스 / 평상시=지오펜스)* 로 정정해야 심사위원(개발자 출신 포함) 앞에서 깨지지 않는다. 그리고 **권한 거부 폴백 + 프라이버시 최소수집**을 더하면, "배터리"를 넘어 "민감 권한 리스크까지 사전 차단"으로 방어 논리가 한 단계 올라간다.

## B. 교체용 섹션

### 5-4. [보조] 기록·커뮤니티 — 바둑(기보·기풍) 테마

- 방문지 사진 수동/자동 업로드, 하루 동선 자동 타임라인, 저녁 **"오늘의 여행 기보(복기)"** 1장 자동 생성(PDF·SNS 카드) → 자연 확산. *(기보 = 바둑 대국 기록 → 하루 여행의 동선·체류·소비를 한 장으로 '복기')*
- **나의 기풍(자기 발견):** 별점·설문 0. **발자국 데이터**로 여행 성향을 바둑 기풍에 빗대어 자동 산정 —
  - 🏃 **속기파** — 유명지를 빠르게 많이 도는 타입(시간당 방문수↑·체류 짧음)
  - 🪨 **두터운 실리파** — 한 곳을 깊고 오래 즐기는 타입(평균 체류↑·소수 정예)
  - ⚔️ **전투형 세력파** — 비주류·신규 지역을 개척하는 타입(비수도권·신규 시군구 비율↑)
- **산정 로직(헛지표 방지):** ①방문 속도 ②평균 체류시간 ③지역 다양성·비주류 비율 — 3축을 정규화해 자동 분류. *데이터가 적으면(5방문 미만) "기풍 산정 중"으로 표기(콜드스타트 방어).*

> 💡 **심사포인트:** "여행만 하면 나의 기풍과 기보가 쌓인다" = **독창성**(발로 두는 바둑) 정조준. 별자리보다 *Stonemark(점령)·리포트(기보)* 와 은유가 정합해 브랜드 일관성↑.
> ⚠️ **문화 접근성:** 바둑 용어가 2030·외국인엔 생소할 수 있어 — **앱 안에선 한 줄 쉬운 설명 병기**("두터운 실리파 = 한 곳을 깊이 즐기는 타입"), **제안서·공공심사 표현은 중립**("여행 성향 분석 리포트"), 외국인 탭은 번역. (§5-8 칭호 정책과 동일 원칙)

### 5-5. 체류 시스템 ★ (생활인구 직결 · 내비급 저전력)

- **체류 자동 인식:** 백그라운드 상시 GPS를 켜지 않고 **OS 융합 위치(Fused Location) + 모션 센서(Activity Recognition)** 를 쓴다. 관광지 100m **Geofence 진입** 시 OS 저전력 타이머를 작동시키고, 모션이 **'정지/도보'를 10분 유지**하며 Geofence를 이탈하지 않았을 때만 **1회 위치 재확인**해 "체류"로 인정. → 배터리 소모를 극소화하며 체류(생활인구) 지표 달성.
- **2모드 설계(핵심):**
  - **가이드 모드 활성**(사용자가 코스를 따라 이동 중) = **포그라운드 서비스 + 지속 알림**(내비와 동일) → Doze 영향 없이 끊김 없는 추적·체크인.
  - **평상시 백그라운드**(앱 닫힘) = **Geofence + 모션 + Significant Location Change** → 거의 0에 가까운 배터리로 진입 이벤트만 감지.
- 한 권역에서 "관광지 + 로컬 맛집 + 숙박"이 모이면 반나절~1박 **체류로 인정**해 가중치 부여.
- **프라이버시 바이 디자인:** 연속 경로는 저장하지 않고 **체크인 이벤트만** 저장. 생활인구 지표는 **익명 집계**, 체크인 좌표는 **6개월 후 자동 파기**(retention 배치 구현 완료).

> 💡 **심사포인트:** 체류 시간/반경 자동 인식을 **내비게이션급 저전력(포그라운드 서비스 + 지오펜스·모션 융합)** 으로 구현해, 위치기반 앱의 고질병인 '배터리 광탈'과 'OS 강제 종료'를 정면 돌파하고 **생활인구(체류인구) KPI를 가장 현실적으로** 달성하는 도구임을 증명. 민감 권한을 쓰면서도 **데이터 최소수집**을 함께 제시 → 신뢰·심사 가점.

### 7-3. 화면별 구현 — 100m 자동 마크 · 체류 (저전력 하이브리드)

| 화면/기능 | 구현 방법 | 난이도·리스크 |
|---|---|---|
| **100m 자동 마크 (백그라운드)** | `react-native-background-geolocation`(transistorsoft)의 **Activity Recognition**으로 정지 시 GPS Off → 배터리 보호, 이동 전환 시에만 Geofence 확인. **OS 한도(iOS 20·Android 100)** 때문에 전국을 다 걸 수 없으므로 **현재 위치 기준 가까운 N개 Geofence만 동적 등록**, 이동하면 Significant Location Change로 깨어나 **재등록(슬라이딩 윈도우)**. | 중 · ⚠️ Expo **dev build** 필요(managed 미지원), **Android 릴리즈 유료 라이선스** |
| **체류(100m·10분)** | 같은 SDK의 **stationary/dwell** 저전력 감지. Android는 Geofence **DWELL(loiteringDelay)** 네이티브, iOS는 region 진입 후 **앱 타이머**로 10분 계산 → 이탈 없으면 1회 핑. | 중~상 · 배터리·정확도 트레이드오프 |
| **가이드 모드 활성 추적** | **포그라운드 서비스 + 지속 알림**(내비 방식)으로 Doze·강제종료 회피, 활성 세션 동안만 고정밀. iOS는 background location(파란 인디케이터). | 중 |
| **권한 거부 폴백** | 'Always'/백그라운드 거부 시 → 가이드 모드 활성 시 **포그라운드 추적 + 수동 "도착했어요" 체크인**으로 기능 유지. 백그라운드 자동화는 *"있으면 더 편한"* 향상 레이어. | 하 (이미 수동 체크인 구현) |

> ☀️ **정정:** "Fused Location으로 Doze를 회피"가 아니라, **활성 세션은 포그라운드 서비스로, 평상시는 Doze 호환 메커니즘(Geofence·FLP·Significant Location Change·고우선 FCM)** 으로 설계하는 것이 정확하다. 상시 백그라운드 GPS는 어떤 방법으로도 배터리·정책을 우회하지 못한다.
> 💡 **심사포인트:** 내비급 저전력 + **권한 거부에도 죽지 않는 폴백** = '배터리 광탈'과 '권한 이탈'이라는 위치기반 앱 2대 리스크를 모두 차단. 점령·체류 KPI는 백그라운드 없이도 달성 가능(백그라운드는 편의 향상).

## D. 비전·브랜드 철학 — "가장 한국적인 것이 가장 세계적인 것이다"

우리의 **한국적 자산이 곧 글로벌 차별점**이다. 이 제품의 한국다움은 글로벌의 *제약*이 아니라 *무기*다.

| 가장 한국적인 것 | → | 가장 세계적인 것 |
|---|---|---|
| **바둑** — 한국·동양의 사고를 담은 게임 | → | 전 세계 어디에도 없는 **"발로 두는 바둑"** 여행 메커닉 (틱톡·구글맵식 흔한 게이미피케이션과 차별) |
| **공공데이터·생활인구·지역화폐** — 한국형 관광 인프라 | → | **데이터로 지방을 살리는 K-관광 모델** 자체가 수출 가능한 IP |
| **로컬·소상공인·진짜 한국** — 서울 너머의 경험 | → | 외국인이 "진짜 한국"을 발견하는 **글로벌 K-로컬 국가대항 랭킹**(§5-7) |

> 한 문장: **"한국을 가장 깊이 즐기게 만드는 도구가, 한국을 세계에 가장 잘 알리는 도구가 된다."**

> ⚠️ **비판적 균형 (배점 정조준 유지):** 본선 배점은 **인바운드 가점 0 / 지역특화(서울 제외) +2**다. 따라서 이 철학은 **브랜드 비전 · V2 글로벌 확장(§5-7, §10 로드맵) · PT 클로징**으로 쓰되, **본선 본문의 무게중심은 끝까지 "지방소멸·체류·지역소비(내국인·지역분산)"** 에 둔다. *북극성 문장이지, 본선 채점표 문장은 아니다.* 글로벌 K-로컬 랭킹은 외국인 실데이터 확보 후 V2 로드맵으로 전개(§5-7·§7-5와 일관).

> 🎤 **PT 클로징 라인:** *"가장 한국적인 것이 가장 세계적인 것입니다. 바둑으로 두는 여행, 데이터로 살리는 지역 — 한국을 가장 깊이 즐기게 만드는 이 도구가, 한국을 세계에 가장 잘 알리는 도구가 됩니다."*
