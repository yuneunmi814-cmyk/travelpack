## 1) 제안 배경 및 필요성

### 현황 — K-관광은 사상 최대 호황
방한 외국인은 2025년 약 1,873만 명으로 역대 최대였던 2019년(1,750만)을 넘어섰고, 2026년 2,036만 명 전망, 정부는 2030년 3,000만 명을 목표로 한다. 국내 여행 수요도 견조하다.

### 문제 — 성장이 '서울'에만 쌓이고, 지방은 비어간다
| 지표 | 수치 | 함의 |
|---|---|---|
| 외국인 방문 편중 | **서울 약 78%** · 부산 16% · 제주 11% | 1극 집중, 다수 지역은 2019년 대비 후퇴 |
| 관광수지 적자 | **약 14.5조 원(2024, 사상 최대)** | 돈이 서울·해외로만 흐름 |
| 지방소멸 | **인구감소지역 89곳** | 소비의 40%를 외부 방문객이 창출(체류인구가 관건) |

### 기존 서비스의 한계 — "인기순"이 쏠림을 키운다
네이버·트리플·구글맵 등은 **인기·검색·평점 순**으로 추천한다. 유명한 곳을 더 유명하게 만들어 **쏠림을 해소하는 게 아니라 강화**한다.

### 필요성 — 분산 + 체류(생활인구)
행안부·한국관광공사의 새 KPI는 **생활인구(체류인구)** 다. 정부·학계의 결론은 *"서울을 거치지 않고 지역에 직접 닿게 하는 것이 지방소멸에 대한 가장 효과적인 관광 솔루션"*. **검색을 뒤집어, '아직 안 가본 곳'을 욕망하게 만드는 도구**가 필요하다.

> 💡 **심사 포인트** · 문제 정의가 한국관광공사 미션(숨은 명소 발굴 · 지역관광 활성화 · 지역 균형발전 · 외국인 만족도)에 **직접 대응** — *제안배경·필요성*에서 명분 확보.

## 2) 서비스 설명

### 핵심 게임 루프 (4동사) — "노력은 0, 결과물은 최대"
```
[코스 고르기] → [코스 따라 걷기 + 가이드] → (도착마다 자동 체크인)
                                              │
                                              ▼ 전부 자동(사용자는 아무것도 안 함)
   • 그 자리에 돌 놓임 → 한국 지도에 내 영토가 칠해짐
   • 비주류 지역일수록 점령 점수 ×N (지역 분산 유도)
   • 하루 하루 동선이 기보(복기)로, 방문지가 감성 엽서로 자동 기록
   • 저녁에 "오늘의 여행 리포트" 1장 자동 생성 → SNS 공유
```
사용자는 **완성형 코스를 고르고 따라 걷기만** 한다. 영토·기록·리포트는 백그라운드로 따라온다.

### 주요 기능
| 기능 | 설명 | 출처 자산 |
|---|---|---|
| 완성형 코스 추천 | 지역·기간·테마별 검증 코스(전국 87개) 3분 준비 | 트래블팩 |
| 가이드 모드 | PostGIS 반경 도착 자동 체크인 + 다음 안내 + 오디오 도슨트 | 트래블팩 |
| 영토 점령 · K-로컬 지수 | 체크인 시 돌이 놓이고 지수 상승, 비주류 지역 가중치 | 스톤마크 |
| 감성 기록 · 리포트 | 동선=기보, 방문지=엽서, 하루=PDF/인스타 카드 자동 | 스톤마크·STARDUST |
| 커뮤니티 | 내 코스·기록 공유, 인기 코스 둘러보기 | 트래블팩 |

### 게이미피케이션 — "K-로컬 지수"
- **K-로컬 지수(단일 지표)** · 체크인할수록 상승, **지방 숨은 명소·전통시장·로컬 소상공인일수록 크게 오름**. 단순 체크인 수가 아닌 **체류 시간·로컬 소비**를 반영해 헛지표를 방지(→ 생활인구 KPI 직결).
- **레벨 칭호** · 앱 안에서는 재미(초보 여행자→…→찐 로컬), 제안서·심사에서는 중립 표현(로컬 탐험가 Lv.1~5)으로 톤 분리.
- **커버리지 랭킹** · 총점이 아니라 **"얼마나 다양한 지역을 갔나"** 기준 → 경쟁이 곧 분산을 유도(점수 파밍 방지).

> 💡 **심사 포인트** · 트래블팩(테스트 88개 · 위치기반서비스 신고 제1175호 · Render 라이브) + 스톤마크(iOS·Android 95%)의 결합 = *서비스 완성도 30점*을 코드로 실증.

## 3) 공공데이터 활용 방안 (★ TourAPI 필수)

| 구분 | API / 데이터 (제공기관: 한국관광공사) | 활용 |
|---|---|---|
| 필수·메인 | **국문 관광정보 서비스(KorService2)** | 코스·관광지 카드, 사진, 상세 |
| 필수·메인 | **위치기반 관광정보** | GPS 도착 인증 → **돌 놓기(체크인)** 핵심 |
| 메인 | **여행코스 정보** | 완성형 코스 골격 |
| **차별화 핵심** | **관광 빅데이터 방문자수(DataLabService)** | **점령 가중치 자동 산정**(비주류 지역 ×N) |
| 차별화 | **카테고리 cat1/cat2 · contentTypeId** | 테마 코스 · 기풍 자동 분류 |
| 글로벌 | **영문 관광정보(EngService2)** | 외국인 3차 타겟 다국어 |
| 부가 | **오디오 가이드(오디·Odii)** | 걸으며 듣는 도슨트 |

> 💡 **심사 포인트** · 방문자수 데이터를 **"지역 분산 가중치"로 융합**해 새 가치를 만든다 = 단순 호출이 아닌 *데이터 활용 20점*. (트래블팩에 `region.visitorScore`로 **이미 적재·동작 중** — 즉시 게임 로직에 연결 가능)

## 4) 서비스의 차별성

### ① 인기순을 뒤집는다
기존 앱이 "유명한 곳"을 보여줄 때, 우리는 **"당신이 아직 안 가본 곳"**을 미점령순으로 욕망하게 만든다. 비주류 지역일수록 보상이 크다.

### ② 진통제 + 비타민 — 다른 위치 게임의 약점을 제거
| | 약점 | 우리 해법 |
|---|---|---|
| 포켓몬GO·Strava | "수집"만 남고 *왜 거기 가야 하나*가 약함 | **완성형 코스**가 가는 이유(진통제) |
| 단순 코스 앱 | 차별성·재방문 약함 | **영토·기록·랭킹**이 독창성·리텐션(비타민) |

→ "게임 점수 때문에 안 가본 곳을 갈까?"라는 행동변화 가정을 **완성형 코스가 원천 제거**한다.

### ③ 경쟁 비교
| | 우리 | 네이버·트리플 | 포켓몬GO |
|---|---|---|---|
| 추천 방향 | **미점령(분산)** | 인기순(쏠림) | 무관 |
| 가는 이유 | 완성형 코스 | 검색 | 캐릭터 |
| 남는 것 | 영토·자기발견 | 기록 없음 | 도감 |

> 💡 **심사 포인트** · "코스 따라가면 한국이 내 바둑판이 된다"는 **글로벌 최초** 결합 = *기획력·독창성 30점*.

## 5) 시장성 및 사업화 방안

### 비즈니스 모델
- **제휴 수수료(B2B)** · 코스 내 로컬 소상공인·숙소·체험 예약 연계
- **프리미엄 코스·패스(B2C)** · 시즌 한정 코스·디자인
- **지자체 협업(B2G)** · 지역 특화 코스·기보 컬렉션 위탁
- **지역화폐 환전(V2)** · 점령·퍼즐 완성 보상을 지역사랑상품권으로
- **IP·데이터 인사이트** · 익명 집계 여행 행동 데이터

### 기대 효과 (관광공사 미션 직접 기여)
- 숨은 명소 발굴(가중치) · 지역 균형(17개 시도) · 외국인 만족(자기발견) · 재방문(평생 못 채우는 점령률)
- **정량 목표(예시, 3년)** · 국내 30만 · 해외 10만 · 월 방문 인증 200만 건 · 비주류 시군구 체류인구 기여

### 단계 로드맵
| 단계 | 시기(예시) | 마일스톤 |
|---|---|---|
| MVP | 본선(10월) | 완성형 코스 + 영토 점령 + K-로컬 지수 + 일일 리포트 |
| V1 | '27 상반기 | **외국인 "글로벌 K-로컬 랭킹"**(국가대항전) · 영문 풀세트 |
| V2 | '27 하반기 | 지역화폐 환전 베타(지자체 파일럿) |

### 지역특화 가점 전략
전국 서비스는 지역특화 가점을 못 받는다. **본선 데모를 강원 등 특정 지역으로 좁히면** 지역특화 가점 +2 & RTO 특별상을 노릴 수 있다(↔ 전국 확장성과 트레이드오프, 팀 결정).

> 💡 **심사 포인트** · 트래블팩의 검증된 BM(제휴 수수료 · 위치기반서비스 신고 완료 · Render 배포)과 게임 리텐션의 결합 = *시장성·사업확장 20~30점*.
