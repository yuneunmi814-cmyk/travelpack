## 4. 통합 개발 우선순위 (PoC 순서)

대부분 자산이 이미 있어 **"새 개발이 아니라 조립"**이 핵심.

1. **트래블팩 가이드 체크인 → 스톤마크 돌 놓기 연결** (둘 다 구현돼 있음 — 이벤트 연결)
2. **`region.visitorScore` → 점령 가중치** (트래블팩에 데이터 이미 적재 = 즉시 가능)
3. **K-로컬 지수 · 커버리지 랭킹** (스톤마크 점령률 로직 재활용)
4. **감성 기록 · 리포트** (STARDUST stars/trips + K-Means 흡수)
5. **영문 경로**(외국인 3차 — EngService2, 트래블팩 i18n 재활용)

> **핵심 리스크 검증** · 7월 PoC에서 "코스→체크인→돌→점령률"이 한 흐름으로 돌면 본선 데모의 80%는 확보. 나머지는 위에 얹는 레이어.
