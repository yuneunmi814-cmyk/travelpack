## 1. 산출물 공유 — 두 개의 웹 MVP
- **축제 큐레이션 웹(윤은미)** · 에어비앤비 레퍼런스로 앞단·검색 완성. 상세 페이지·큐레이션은 미완(자료 덤핑 단계). 데이터 소스 = 한국관광공사·서울 열린데이터광장·문화공공데이터광장 등.
- **전국 맛집(식당) 웹(최용우)** · 관광공사 API + 카카오, **실데이터**. 검색은 시군구·동 단위까지, 상세에 주소·영업시간·휴무일·전화·길찾기·교통(모달). API 호출 한도(개발계정 1,000건) 때문에 **운영계정 신청** 중.
- 둘 다 **큐레이션 미적용**이라 지금은 디테일 손대지 말고, 타사 기능 정리 후 **한꺼번에 세팅**하기로.

## 2. 개발 협업 방식 — 1인 개발/배포 + 1인 디버깅
- **윤은미** = 웹 개발·배포 집중(둘 다 바이브 코딩이라 중복 불필요). **최용우** = 버그 리포트(전체 화면 캡처 → 디스코드, **한 번에 모아서**) + 점검. **전도준** = 검수·완료 확인 순환.
- 두 웹은 **기능이 많은 윤은미 것 기준으로 통합**(불필요 기능은 빼는 게 쉬움). 최용우의 식당 작업 **로그·프롬프트를 raw(MD) 파일로 공유** → 윤은미가 흡수.
- 방향성: 별도 네이티브 앱 대신 **앱처럼 최적화한 웹(웹앱)**. 모바일 반응형(가로 슬라이드 → 세로 스크롤) 필수.

## 3. 큐레이션·노출 설계 (전도준 레퍼런스 분석)
- **첫 화면 큐레이션** · 에어비앤비식 이미지 그리드는 데이터 누적 전엔 안 예쁨 → **지역(예: 서울) 진입 시 상단 = 월별 메인 축제(예쁜 노출 이미지), 하단 = 후기·블로그(현장감, 데이터 쌓이면 상위 노출)** 방식 유력.
- **편의시설 아이콘(상세 상단)** · 화장실 · 장애인/휠체어 대여 · 반려동물 동반 · 키즈존/노키즈 · **직행/셔틀버스 유무** 등 — 관광객 편의 직관 표시.
- **SNS·커뮤니티 연동** · 내 피드/공유 버튼. 태국=**Pantip(판팁)** > 인스타 > 틱톡, 일본=**4travel(포트래블)** 정보수집. → 페이스북·인스타·틱톡 채널 개설.
- **태그 추천·인기태그** · 몰아주기로 **SEO 최적화**.
- **교통 큐레이션** · Uber/구글맵식 심플. **내 위치(GPS) 기반 거리·이동 자동 계산**(축제↔식당↔현위치 드래그 연동). *지역 이동의 핵심 불안은 교통* — 식당보다 교통 안내가 우선.
- **맞춤 큐레이션(향후)** · 첫 진입 시 국적·SNS 선택값 수집 → 국가별 디자인·노출 차별화. 단 난이도 높아 **데이터 쌓인 뒤로 보류**, 일단 공통 화면.

## 4. 타겟 확정 — 태국 1순위, 일본 차순위
- 동남아 세분화(매출 기준): **싱가포르**=고급·5성/크루즈(향후), **베트남**=단체관광 B2B(향후 노다지), **태국**=여행 성숙도 높음·**자유여행·긴 체류·K-pop 선호·B2C 적합 → 1순위.** **일본**=안정형·정보수집 많음(노가다 필요)·구글맵 별표 활용.
- **노출 채널** · 태국 = Pantip(웹·GEO 가능)·페이스북·인스타(이미지)·화려한 틱톡 / 일본 = 인스타 이미지·4travel.
- 초기 마케팅 비용이 없으니 **한국 친화 콘텐츠(K드라마 촬영지 등) 트래픽을 감지**해 상위 큐레이션.

## 5. 지도 — 구글맵 공백이 기회
- 외국인은 구글맵을 쓰는데 **한국은 안보 이슈로 구글에 상세 지도·내비 데이터 미제공**(국내 카카오/네이버만 보유). → **우리 웹에서 목적지를 연동**(카카오 로그인 불필요·클릭 즉시 표시)하는 게 외국인에겐 강한 경쟁력. 일본인은 구글맵 별표를 많이 써서 구글맵도 병행 검토.

## 6. 다국어 — 태국어·일본어 우선
- 영어가 아닌 **태국어·일본어 직관 표기**(국기/언어를 드롭다운 말고 바로 노출). 타겟 최적화. (성형·뷰티 다국어 의료관광 수요는 향후.)

## 7. 서비스명·브랜드 확정 — 코타(KOTA)
- 각자 후보 3~4개 제출 → 투표. **K-네이밍(케이~)은 한류 유행에 함께 휘발될 위험** 지적.
- **결정**
  - 정식명 **Korea Festa**, 약칭·브랜드 **코타(KOTA = Korea Tiger)**.
  - **마스코트 = 호랑이 "코타"** (AI가 호랑이 무늬를 못 만들어 오히려 IP/라이선스 확보에 유리).
  - **로고·캐릭터 제작 = 전도준**(레퍼런스 기반), 영문 표기.
